# Work Log

## Purpose

This document provides a rolling monthly log of work completed, ongoing work, and planned work.

## 🧠 This Is Your Memory

> **This is your promo log + evidence for leadership.**

This file replaces chat memory. Cursor can regenerate summaries and status updates from this file reliably. When you need to understand what work was done, what's in progress, or generate leadership updates, reference this file - it contains the complete work history.

**Key Principle**: Document work as it happens. This enables conversation tracking and reliable status generation without relying on chat memory.

## How to Use This File

### For Conversation Tracking

When starting a new conversation or session:
1. **Read this file first** to understand current work status
2. **Reference specific work items** by date or topic when needed
3. **Update this file** immediately after completing work or changing status

### For Leadership Updates

This file provides evidence for:
- Progress reports
- Status updates
- Achievement summaries
- Risk identification
- Resource needs

**Prompt for Cursor**: "Read `09_DECISIONS_WORKLOG/WORKLOG.md` and generate a leadership update for [period]."

### For Regenerating Context

Cursor can use this file to:
- Understand what work has been completed
- Generate status summaries
- Identify ongoing work and blockers
- Create progress reports
- Provide evidence for performance reviews

**Rule**: Update this file regularly (at least weekly). The more current it is, the more reliable the generated summaries will be.
