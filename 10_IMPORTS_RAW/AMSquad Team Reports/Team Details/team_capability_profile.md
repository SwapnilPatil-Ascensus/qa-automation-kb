## 1. Overview

The QA Automation team is a **five-person** squad covering **Prime V2 and V3 UI automation**, **API automation**, **performance testing**, and **pipeline / cross-program coordination**. Execution strength is concentrated in **Java/Selenium/Cucumber/TestNG** UI suites and **REST API** validation; **performance** is anchored in **JMeter**-oriented delivery with Jenkins integration. **Pipeline strategy, intake, and multi-team enablement** are led at the program level.

**Coverage areas**

| Area | Snapshot |
|------|----------|
| **UI automation** | Strong — dual track: **V2** (legacy/member/CSR-heavy) and **V3** (UE, IDP, CSR). |
| **API automation** | Strong and **growing** — dedicated tech-lead ownership; CI pipeline milestones in flight. |
| **Performance testing** | **Concentrated** — primary dedicated engineer; supported by broader team for env/auth dependencies. |
| **Pipeline / DevOps integration** | **Growing** — program-led; GitHub Actions / GitLab direction under evaluation per leadership updates. |

**Overall:** The team is **strong in regression execution and suite growth** on V2/V3, with **clear depth in API and performance** where staffed; **strategy, backlog, and pipeline expansion** are actively owned alongside hands-on delivery.
