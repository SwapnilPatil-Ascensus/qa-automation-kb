## Root cause

 

`helm upgrade --install` failed with:

 

```text
Secret "enrollment-core-services-credential-v1" in namespace "agsupaws-useast1-stage1" exists and cannot be imported into the current release: invalid ownership metadata; label validation error: missing key "app.kubernetes.io/managed-by": must be set to "Helm"; annotation validation error: missing key "meta.helm.sh/release-name": must be set to "agsup-ui-enrollment"; annotation validation error: missing key "meta.helm.sh/release-namespace": must be set to "agsupaws-useast1-stage1"
```

 

So:

 

1. That Secret **already existed** in `agsupaws-useast1-stage1`.
2. It was **not** created by this Helm release (no Helm `managed-by` label / `meta.helm.sh/*` annotations).
3. After your chart change, Helm **now tries to manage** that object as part of `agsup-ui-enrollment`, and Helm **will not take over** a Secret that wasn’t created by the release.

 

That lines up with the earlier analysis: stage1 used to **not** get this Secret from Helm, so it was almost certainly **created some other way** (script, copy-paste, another pipeline). QC4 worked because that Secret was always Helm-owned there.

 

## What to do (ops)

 

Pick one (platform / someone with kube access on `unite-eks-stage-useast1`):

 

**A – Simplest (usually):** Delete the orphan Secret, then rerun the deploy so Helm recreates it with correct ownership and the new data:

 

```bash
kubectl delete secret enrollment-core-services-credential-v1 -n agsupaws-useast1-stage1
# then re-run deploy:passive (or let the pipeline retry)
```

 

Expect a short window where new pods might not mount the secret until the upgrade completes (often acceptable for a passive/preview slot).

 

**B – “Adopt” into the release:** Patch the existing Secret so it has the label/annotations Helm expects, **then** run `helm upgrade` again. This is easy to get wrong (checksums, release history); many teams prefer **A**.

 

## Not a chart bug (by itself)

 

Your Helm change is doing the right thing for stage1; the failure is **collision with a pre-existing, non-Helm Secret**. After it’s removed or properly adopted, the same chart version should install.
