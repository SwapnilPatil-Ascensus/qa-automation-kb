When a CSR user navigates to Profile & Documents → Authorized Individual on an ABLE plan account and clicks “Edit Address, Phone or Identity”, the application displays a generic error page instead of opening the edit form. This blocks Authorized Individual (AI) maintenance for ABLE accounts.

Modules affected (hub index):

stage1-csr-acct-maintenance  

Enrollment technical detail (sample — Idaho):

Error! please try again later.

Generic error page:

ErrorCode[This functionality is not available at this time...], [ts3], [06/05/2026 0:40:18]

Steps to Reproduce

Log in to CSR for an ABLE plan (e.g. NYB, AKB, LAB).

Jump to a member account that has Authorized Individual information.

Navigate to View Profile and Documents.

Click Authorized Individual.

On the Authorized Individual summary page, click “Edit Address or Entity”
(or “Edit Address, Phone or Identity” for person-type AI).

Observe the result.

Expected Result

The Authorized Individual edit screen loads, allowing the CSR user to update address, phone, identity, and/or entity details, then proceed with Next / Submit.

Actual Result

Error! please try again later.
Your request cannot be completed at this time. Please contact Plan administrator.
ErrorCode[This functionality is not available at this time. Please try again later.]
