# Glossary

## Purpose

This document defines key terms, acronyms, and concepts used throughout the QA automation knowledge base.

## A

**ADR** - Architecture Decision Record. A document capturing an important architectural decision.

**API** - Application Programming Interface. A set of protocols and tools for building software applications.

## B

**BDD** - Behavior-Driven Development. A software development approach that uses natural language to describe software behavior.

## C

**CI/CD** - Continuous Integration/Continuous Deployment. Practices that automate the integration and deployment of code changes.

**Cucumber** - A BDD framework that supports behavior-driven development using Gherkin syntax.

## D

**Daily Regression** - Automated test suite executed daily to verify core functionality remains intact.

## E

**Environment** - A specific instance of the application (e.g., Stage1, Stage2, PRO, QA, DR).

## F

**Flakiness** - Tests that pass or fail inconsistently without code changes.

## G

**Gherkin** - A business-readable domain-specific language used in Cucumber for writing test scenarios.

## I

**Integration Testing** - Testing that verifies the interaction between different components or systems.

## J

**Jenkins** - An open-source automation server used for CI/CD pipelines.

**Jira** - A project management and issue tracking tool.

## M

**Maven** - A build automation tool used primarily for Java projects.

## P

**Performance Testing** - Testing to determine system responsiveness, stability, and scalability under a workload.

**POM** - Page Object Model. A design pattern for organizing test automation code.

## Q

**qTest** - A test management tool used for organizing and tracking test cases and results.

## R

**RCA** - Root Cause Analysis. A process for identifying the underlying cause of a problem.

**REST Assured** - A Java library for testing RESTful APIs.

**Release Regression** - Comprehensive test suite executed before a production release.

## S

**Selenium** - An open-source tool for automating web browsers.

**Smoke Test** - A minimal set of tests that verify basic functionality.

**Sanity Test** - A focused test suite that verifies specific functionality after changes.

**Stage1/Stage2** - Staging environments used for testing before production.

## T

**TestNG** - A testing framework for Java inspired by JUnit and NUnit.

**Triage** - The process of prioritizing and categorizing test failures or defects.

## U

**UI Automation** - Automated testing of user interfaces.

## V

**Version Control** - A system that tracks changes to files over time (e.g., Git).
