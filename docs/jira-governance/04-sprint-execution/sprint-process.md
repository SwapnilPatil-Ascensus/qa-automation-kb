# Sprint Process

## 1. Sprint parameters (set by team)

| Parameter | Example | WHO sets |
|-----------|---------|----------|
| Length | 2 weeks | Team charter |
| Sprint name | `Sprint 24 - 2026-03-10` | SM |
| Goal | One sentence | PO + Team at planning |

## 2. Lifecycle (WHEN)

| Phase | WHEN | Primary doc |
|-------|------|-------------|
| Refinement | Mid-sprint weekly | [refinement-process.md](refinement-process.md) |
| Planning | Day 1 | [sprint-planning-checklist.md](sprint-planning-checklist.md) |
| Execution | Days 2–N | Jira board + [daily-standup-script.md](daily-standup-script.md) |
| Review | Last day | Demo agenda (team template) |
| Retro | Last day | Retro board (Jira/Confluence/Miro) |

## 3. WHO — core roles

| Role | During sprint |
|------|----------------|
| PO | Answers scope questions; accepts/rejects |
| SM | Removes impediments; protects process |
| Team | Delivers; updates Jira daily |
| Tech Lead | Unblocks design; reviews risk |

## 4. Scope change rules

| Change | Allowed | WHO approves |
|--------|---------|--------------|
| Add Story mid-sprint | Only if same size removed or PO drops another | PO |
| Expand AC | New Story or spike | PO |
| Emergency insert | Replace lowest priority item | PO + Tech Lead |

## 5. Sprint cancellation

| Trigger | Action |
|---------|--------|
| >50% scope invalidated | PO calls cancel; SM documents; re-plan |

Rare; log in Confluence + Jira sprint comment.
