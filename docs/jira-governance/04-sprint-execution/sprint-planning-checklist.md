# Sprint Planning Checklist

## 1. WHEN

| Item | Timing |
|------|--------|
| Duration | 2–4 hours for 2-week sprint (team-calibrated) |
| Participants | Whole team + PO; Tech Lead; SM facilitates |

## 2. Pre-planning (WHO: SM + PO | WHEN: day before)

| # | Check | ✓ |
|---|-------|---|
| P1 | **Velocity** / capacity known (PTO, holidays) | |
| P2 | **Ready** filter: top items meet [DoR](../03-story-standards/definition-of-ready.md) | |
| P3 | **Dependencies** on external teams flagged | |
| P4 | Board columns reflect workflow | |

## 3. Planning agenda (enforce order)

| Step | WHAT | WHO | Time |
|------|------|-----|------|
| 1 | **Sprint goal** proposal from PO | PO | 10m |
| 2 | **Capacity** confirmation | Team | 10m |
| 3 | Walk **Ready** backlog top-down; pull Stories | Team | bulk |
| 4 | **Task** breakdown for each Story (optional subtasks) | Assignees | per Story |
| 5 | **Commit** — team says “we commit to goal + these keys” | Team | 10m |
| 6 | **Risks** logged on sprint or Epic comment | SM | 5m |

## 4. Outputs (must exist)

| Output | WHERE |
|--------|--------|
| Sprint goal | Jira sprint field + Confluence optional |
| Committed Stories | Sprint backlog only |
| Unselected top items | Still ranked in backlog |

## 5. Checklist — per Story committed

| # | Item | ✓ |
|---|------|---|
| C1 | DoR satisfied | |
| C2 | Assignee (or pair noted in comment) | |
| C3 | Linked Epic / version | |
| C4 | No hidden dependency | |

## 6. Anti-patterns (reject)

| Anti-pattern | Fix |
|--------------|-----|
| PO assigns all work | Team pulls and owns |
| “We’ll figure out AC in sprint” | Defer Story |
| 100% utilization assumed | Use capacity buffer (~10–15%) |
