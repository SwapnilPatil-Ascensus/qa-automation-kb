# Refinement Process (Backlog Grooming)

## 1. Objective

Make upcoming Stories **Ready** (DoR) and **estimated** without starting implementation.

## 2. WHEN

| Cadence | Default |
|---------|---------|
| Weekly | Mid-sprint, 60–90 min |
| Extra | If planning is in <3 days and Ready queue low |

## 3. WHO

| Role | Participation |
|------|----------------|
| PO | Presents priority order; clarifies “why” |
| Team | Asks questions; estimates |
| Tech Lead | Flags design, risk, splitting |
| QA | Testability, AC quality |
| SM | Facilitates; timeboxes |

## 4. Agenda (template)

| Segment | Duration | Output |
|---------|----------|--------|
| Review **top 5–10** backlog items | 45m | Questions logged |
| **Split** large items | 15m | New Stories linked |
| **Estimate** | 20m | Points on Stories |
| **Dependency** pass | 10m | Owners + dates |

## 5. Entry / exit

| Gate | Rule |
|------|------|
| Enter refinement | Item is ranked in top backlog slice |
| Exit as **Ready** | DoR satisfied ([definition-of-ready.md](../03-story-standards/definition-of-ready.md)) |

## 6. Estimation rules

| Rule | Enforcement |
|------|-------------|
| Whole team estimates (or delegated squad) | Facilitator enforces |
| Re-estimate if scope changes >30% | Same session |

## 7. Parking lot

Items exceeding timebox → comment `refinement-parking-lot` + next session first.
