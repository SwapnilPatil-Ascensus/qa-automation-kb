# Sprint Reporting

## 1. WHEN

| Report | WHEN produced | WHO |
|--------|---------------|-----|
| Sprint outcome | Last day of sprint | SM compiles; PO approves |
| Velocity | End of sprint | SM |
| Carry-over analysis | First day next sprint | PO + Team |

## 2. Minimum content (enforced)

| Section | Source |
|---------|--------|
| Sprint goal | Jira sprint field |
| Planned vs completed keys | Jira sprint report |
| Velocity (points) | Sprint report |
| Carry-over + reason | Comment on carried Stories |
| Impediments | Jira + retro notes |

## 3. Template — sprint outcome (Confluence or email)

```markdown
## Sprint [N] ([dates])
**Goal:** …
**Completed:** PROJ-… (link filter)
**Not done:** PROJ-… — reason …
**Velocity:** X / Y points
**Incidents during sprint:** …
**Notes:** …
**Dashboard:** [URL]
```

## 4. Jira native

Use **Reports → Sprint Report** and **Burndown**; export or screenshot for archive if policy requires.

## 5. Governance

| Rule | Enforcement |
|------|-------------|
| Numbers match Jira | No hand-edited counts |
| Carried work has reason | SM blocks empty carry |
